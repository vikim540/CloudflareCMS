# 數據庫設計

> 本文件定義了基於 PbootCMS 3.2.12 數據庫結構的 Cloudflare D1 數據庫設計。所有表結構保持與 PbootCMS 原版完全一致,僅在 Go 版基礎上新增了少量擴展表,並通過冪等索引提升查詢效能。

---

## 一、設計原則

### 1.1 數據庫零改動

PbootCMS 原版的所有表結構和字段**不做任何修改、不刪除任何字段**。這是硬約束,確保從 PbootCMS 遷移過來的數據可以無縫對接。

| 規則 | 說明 |
|------|------|
| 不修改字段 | 原版字段的類型、長度、預設值均保持不變 |
| 不刪除字段 | 即使某些字段在本項目中不使用（如會員相關字段）,也予以保留 |
| 不重命名字段 | 原版字段名稱完全保留,包括命名風格不一致的情況（如 `ay_single` 的 `createtime` 無下劃線） |
| 表前綴不變 | 所有表保持 `ay_` 前綴 |

### 1.2 僅允許新增表

Go 版已在 PbootCMS 原版基礎上新增了以下 5 張表,本項目沿用:

| 新增表 | 用途 |
|--------|------|
| `ay_area` | 區域/站點管理（多站點隔離,本項目固定使用 `cn`） |
| `ay_role_area` | 角色-區域關聯映射 |
| `ay_301_redirect` | 301 重定向規則（SEO 用） |
| `ay_media_mark` | 媒體水印標記配置 |
| `ay_content_ext` | 內容擴展字段存儲（配合 `ay_extfield` 動態字段） |

### 1.3 僅允許冪等添加索引

所有索引使用 `CREATE INDEX IF NOT EXISTS` 語法,確保可以安全重複執行:

```sql
CREATE INDEX IF NOT EXISTS idx_content_filename ON ay_content(filename);
```

### 1.4 SQLite / D1 兼容語法

- 所有 `CREATE TABLE` 使用 `IF NOT EXISTS`
- 主鍵使用 `INTEGER PRIMARY KEY AUTOINCREMENT`
- 不使用 MySQL 專有語法（如 `ENGINE=InnoDB`、`CHARSET`）
- 字段類型使用 SQLite 親和類型: `INTEGER`、`TEXT`、`REAL`
- 時間戳存儲為 `INTEGER`（Unix 時間戳）或 `TEXT`（`YYYY-MM-DD HH:MM:SS` 格式）

| last_login_ip | TEXT | '' | 最後登錄 IP |
| lastlogintime | INTEGER | 0 | 最後登錄時間（Unix 時間戳） |

CREATE TABLE IF NOT EXISTS ay_user (

## 四、D1 適配注意事項

### 4.1 D1 與 SQLite 的語法差異

D1 基於 SQLite 引擎,大部分語法完全兼容,但存在以下限制:

| 特性 | SQLite | D1 | 說明 |
|------|--------|-----|------|
| `CREATE TABLE` | 完整支持 | 支持 | 使用 `IF NOT EXISTS` |
| `ALTER TABLE ADD COLUMN` | 支持 | 支持（有限） | 不能添加 `NOT NULL` 無預設值的列 |
| `ALTER TABLE DROP COLUMN` | 支持（3.35+） | 不支持 | D1 不支持刪除列 |
| `ALTER TABLE RENAME COLUMN` | 支持（3.25+） | 不支持 | D1 不支持重命名列 |
| `WITH RECURSIVE` (CTE) | 支持 | 支持 | 用於遞歸查詢欄目樹 |
| `INSERT OR REPLACE` | 支持 | 支持 | 冪等插入 |
| 事務 | `BEGIN/COMMIT` | `batch()` API | D1 使用 `db.batch()` 實現事務 |
| `ATTACH DATABASE` | 支持 | 不支持 | D1 不支持附加數據庫 |
| 觸發器 | 支持 | 不支持 | D1 不支持觸發器,需在應用層實現 |
| 視圖 | 支持 | 不支持 | D1 不支持視圖,需在應用層實現 |

### 4.2 遞歸 CTE 查詢子孫欄目

D1 支持 `WITH RECURSIVE` 語法,可用於查詢某個欄目下的所有子孫欄目:

```sql
-- 查詢欄目 scode='1' 下的所有子孫欄目
WITH RECURSIVE descendant_sorts AS (
    -- 基礎查詢: 直接子欄目
    SELECT id, scode, pcode, name, sorting, status
    FROM ay_content_sort
    WHERE pcode = '1' AND status = 1

    UNION ALL

    -- 遞歸查詢: 子欄目的子欄目
    SELECT s.id, s.scode, s.pcode, s.name, s.sorting, s.status
    FROM ay_content_sort s
    INNER JOIN descendant_sorts d ON s.pcode = d.scode
    WHERE s.status = 1
)
SELECT * FROM descendant_sorts ORDER BY sorting;
```

### 4.3 D1 batch() 事務操作

D1 的 `batch()` 方法將多個語句作為一個事務執行,保證原子性。在 Rust `worker` crate 中:

```rust
use worker::D1Database;

// 批量插入內容（事務性操作）
let statements = vec![
    db.prepare("INSERT INTO ay_content (title, scode, ...) VALUES (?, ?, ...)")?,
    db.prepare("INSERT INTO ay_content_ext (contentid, ext_1, ...) VALUES (?, ?, ...)")?,
];

let results = db.batch(statements).await?;
// 所有語句要麼全部成功,要麼全部回滾
```

### 4.4 D1 免費版限制

| 指標 | 免費版限制 | 應對策略 |
|------|-----------|---------|
| 每日行寫入 | 100,000 行/天 | 操作日誌僅記錄關鍵操作（登錄、內容發布/刪除、配置修改） |
| 每日行讀取 | 5,000,000 行/天 | 熱點配置緩存到 KV,減少 D1 查詢 |
| 單次查詢返回行數 | 5,000 行 | 分頁查詢,默認每頁 20 條 |
| 數據庫大小 | 500 MB | 定期清理系統日誌 |
| 單次查詢超時 | 10 秒 | 優化索引,避免全表掃描 |

### 4.5 KV 緩存策略

以下高頻讀取、低頻修改的數據應緩存到 KV,減少 D1 查詢壓力:

| 數據類型 | KV Key 格式 | TTL | 失效觸發 |
|---------|------------|-----|---------|
| 系統配置 | `config:all` | 1 小時 | 配置修改時主動刪除 |
| 站點信息 | `site:cn` | 1 小時 | 站點信息修改時主動刪除 |
| 公司信息 | `company:cn` | 1 小時 | 公司信息修改時主動刪除 |
| 欄目樹 | `sort:tree:cn` | 30 分鐘 | 欄目增刪改時主動刪除 |
| 後台菜單 | `menu:all` | 1 小時 | 菜單修改時主動刪除 |
| 301 重定向規則 | `redirect:all` | 10 分鐘 | 規則修改時主動刪除 |
| 自定義標籤 | `label:all` | 1 小時 | 標籤修改時主動刪除 |

### 4.6 D1 訪問模式（Rust worker crate）

```rust
use worker::{D1Database, Result};

// 查詢多行
let stmt = db.prepare("SELECT * FROM ay_content WHERE scode = ?1 AND status = 1 ORDER BY sorting LIMIT ?2 OFFSET ?3")?;
let result = stmt.bind(&[&scode, &limit, &offset])?.all().await?;
let contents: Vec<Content> = result.results()?;

// 查詢單行
let stmt = db.prepare("SELECT * FROM ay_content WHERE id = ?1")?;
let content: Option<Content> = stmt.bind(&[&id])?.first().await?;

// 寫入（INSERT/UPDATE/DELETE）
let stmt = db.prepare("UPDATE ay_content SET title = ?1, update_time = ?2 WHERE id = ?3")?;
let result = stmt.bind(&[&title, &now, &id])?.run().await?;
// result.meta.changes 表示受影響的行數
```

## 六、數據庫遷移說明

### 6.1 從 PbootCMS SQLite 遷移

PbootCMS 默認使用 SQLite 數據庫,遷移到 D1 的步驟:

1. **導出原數據庫**: 使用 SQLite 工具導出 PbootCMS 的 `.db` 文件為 SQL 腳本
2. **上傳到 D1**: 使用 `wrangler d1 execute` 命令導入 SQL 腳本

```bash
# 創建 D1 數據庫
pnpm wrangler d1 create cms-db

# 執行建表 SQL
pnpm wrangler d1 execute cms-db --file=./schema.sql

# 導入 PbootCMS 原始數據
pnpm wrangler d1 execute cms-db --file=./pbootcms_data.sql
```

### 6.2 遷移注意事項

| 事項 | 說明 |
|------|------|
| 表結構 | 直接使用 PbootCMS 原版表結構,無需轉換 |
| 數據導出 | 使用 `sqlite3 .dump` 或 DB Browser for SQLite 導出 |
| 字符編碼 | 確保 SQL 文件為 UTF-8 編碼 |
| 索引創建 | 導入數據後再執行索引創建腳本,避免插入時索引開銷 |
| Go 版新增表 | 遷移 PbootCMS 數據後,單獨執行 Go 版新增表的建表語句 |
| 管理員密碼 | PbootCMS 原版密碼為雙 MD5,遷移後可直接使用 |
| 序列值 | SQLite 的 `sqlite_sequence` 表記錄了 AUTOINCREMENT 的當前值,遷移時需一併導入 |

### 6.3 遷移驗證

```sql
-- 驗證表數量（應為 32 張）
SELECT count(*) as table_count FROM sqlite_master WHERE type='table' AND name LIKE 'ay_%';

-- 驗證內容數量
SELECT count(*) as content_count FROM ay_content;

-- 驗證欄目數量
SELECT count(*) as sort_count FROM ay_content_sort;

-- 驗證管理員賬號
SELECT id, username, realname, status FROM ay_user;

-- 驗證索引
SELECT name FROM sqlite_master WHERE type='index' AND name LIKE 'idx_%';
```

## 附錄: 完整建表腳本執行順序

建議按以下順序執行建表腳本,確保外鍵引用的表先創建:

```
1. ay_config        （系統配置,無依賴）
2. ay_site          （站點信息,無依賴）
3. ay_company       （公司信息,無依賴）
4. ay_model         （內容模型,無依賴）
5. ay_content_sort  （欄目分類,依賴 ay_model 的 mcode）
6. ay_content       （內容表,依賴 ay_content_sort 的 scode）
7. ay_content_ext   （內容擴展,依賴 ay_content 的 id）
8. ay_extfield      （擴展字段定義,依賴 ay_model 的 mcode）
9. ay_single        （單頁表,依賴 ay_content_sort 的 scode）
10. ay_menu         （後台菜單,無依賴）
11. ay_menu_action   （菜單操作,依賴 ay_menu 的 mcode）
12. ay_role          （角色表,無依賴）
13. ay_role_level    （角色權限,依賴 ay_role 的 rcode）
14. ay_user          （管理員表,依賴 ay_role 的 rcode）
15. ay_message       （留言表,無依賴）
16. ay_form          （表單表,無依賴）
17. ay_form_field    （表單字段,依賴 ay_form 的 fcode）
18. ay_link          （友情連結,無依賴）
19. ay_slide         （幻燈片,無依賴）
20. ay_tags          （標籤表,無依賴）
21. ay_label         （自定義標籤,無依賴）
22. ay_syslog        （系統日誌,無依賴）
23. ay_301_redirect  （301重定向,無依賴）
24. ay_area          （區域表,無依賴）
25. ay_role_area     （角色區域,依賴 ay_role 和 ay_area）
26. ay_media_mark    （水印配置,無依賴）
27-30. 會員系統表     （ay_member, ay_member_group, ay_member_field, ay_member_comment）
31. ay_database      （數據庫備份,無依賴）
32. ay_type          （類型字典,無依賴）
```

> **注意**: SQLite 不強制外鍵約束（默認關閉）,上述順序僅為邏輯清晰。實際執行時可一次性導入全部建表語句。
